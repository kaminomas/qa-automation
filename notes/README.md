# 学习笔记

记录 QA 自动化学习过程中的知识点、踩坑记录和最佳实践。

## 组织方式
每个主题建立独立的 Markdown 文件，例如：

| 文件 | 内容 |
|------|------|
| `web3-rpc.md` | 以太坊 JSON-RPC 接口笔记 |
| `contract-testing.md` | 智能合约测试技巧 |
| `performance-tuning.md` | 性能调优记录 |
| `tools-comparison.md` | 测试工具对比 |

## 写作规范
- 记录**问题背景** → **解决过程** → **结论**，便于日后回溯
- 代码片段附上可运行的最小示例
- 打上标签方便检索（`#web3` `#performance` `#api`）
